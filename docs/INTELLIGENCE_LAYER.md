# Intelligence Layer — inno413

## Messy Inputs
- Free-text design notes ("football jerseys with our logo, red trim, white number")
- Uploaded image files (design artwork of varying quality)
- Mixed size requests entered by customer

## Auto-Structure (v1 rule-based, no LLM needed)
```json
{
  "order_id": "uuid",
  "urgency_flag": "high",
  "urgency_reason": "needed_by within 7 days",
  "design_completeness_score": 0.6,
  "missing_fields": ["design_file"],
  "total_quantity": 22,
  "size_distribution": {"S":4, "M":8, "L":6, "XL":4}
}
```

## Events to Track
- Order submitted
- Design file uploaded / missing
- Deposit paid / unpaid after 24h
- Order status changed
- Invoice issued and unpaid >3 days

## Scoring Rules (rule-based v1)
- **Urgency score:** needed_by ≤ 7 days → high; ≤ 14 days → medium; else low
- **Completeness score:** +0.25 each for design file, design notes, delivery address, size breakdown filled
- Scores stored as plain numeric columns; no ML in v1

## What Gets Ranked
- Staff dashboard default sort: urgency score desc, then created_at asc
- Overdue deposits surfaced at top with badge

## AI Field (v1 → later)
- `orders.ai_summary`: 1-sentence natural language brief auto-generated by GPT after submission
- Stored with source, confidence, review_status; staff can approve/reject
- v1: leave blank or trigger manually; later: auto on order_submitted event
